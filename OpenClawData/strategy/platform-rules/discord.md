# Discord — Platform Rules

## Content Types
- Announcements (product updates, launches)
- Discussion starters (AI topics, India problems)
- Build logs (what was built today)
- Community questions and polls
- Resource sharing

## Tone
- Community-friendly, approachable
- Technical depth welcome (developer audience)
- Casual but informative
- Responsive to community questions

## Format Rules
- Markdown formatting supported
- Use embeds for structured announcements
- Keep messages scannable (bullets, headers)
- Use channels appropriately (announcements vs general)

## Content Priorities
- Product launch/update announcements
- Behind-the-scenes build updates
- AI tool recommendations
- Community discussions on India AI
- Open source contribution highlights
- Bug fixes and feature requests

## Approval Level
- L1 (auto-approved) for owned server announcements
- L2 for community engagement posts
- L3 for anything posted in external servers

## Posting Cadence
- Announcements: as needed (2-3/week)
- Community engagement: daily
- Build logs: 2-3/week

## Restrictions
- No spam across multiple servers
- No self-promotion in other communities without genuine contribution
- No DM marketing
- Auto-posting only to owned server
