# Feature Highlight Template

## Objective
Show one specific capability of {product} and why it matters to the audience.

## Hook Styles
- "Here's something {product} can do that most people don't know about."
- "{Feature} sounds simple. But for {audience} in India, it changes everything."
- "We just shipped {feature}. Here's why it matters."

## Structure
1. **Hook**: Feature name or capability, framed as value
2. **Context**: Why this feature exists (the problem it addresses)
3. **How it works**: 1-2 sentence explanation, no jargon
4. **Impact**: What changes for the user
5. **CTA**: Try it, see it, learn more

## Image Style
- UI screenshot showing the feature
- Before/after comparison
- Simple diagram of how it works

## Safe Claim Rules
- Feature must actually exist in the current product
- No performance claims without data
- "In development" features must be labeled as upcoming
