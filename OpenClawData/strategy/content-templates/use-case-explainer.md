# Use Case Explainer Template

## Objective
Show how {product} solves a specific problem for a specific audience. Concrete, not abstract.

## Hook Styles
- "{Audience} spends {time/effort} on {problem}. Here's a better way."
- "What if {specific outcome} was possible with AI? It is."
- "A {role} in {context} needed {solution}. Here's what we built."

## Structure
1. **The person**: Who has this problem (specific role/context)
2. **The problem**: What they struggle with (concrete, relatable)
3. **The solution**: How {product} addresses it (specific features)
4. **The outcome**: What changes (measurable or visible)
5. **CTA**: Try it for your situation

## Rules
- Use case must be genuinely possible with current product capabilities
- Don't fabricate user stories — use "imagine" or "consider" framing
- Outcomes must be realistic, not inflated
- Include "in development" disclaimer if feature isn't live yet
