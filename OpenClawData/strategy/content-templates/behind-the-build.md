# Behind the Build Template

## Objective
Share real development progress, technical decisions, and building challenges transparently.

## Hook Styles
- "Here's what I built this week. And what broke."
- "A decision I made about {technical choice}. Here's my reasoning."
- "Day {N} of building {product}. Today's progress:"

## Structure
1. **What was built/changed**: Specific, concrete
2. **Why**: The reasoning or user need behind it
3. **Challenge**: What was hard or surprising
4. **Result**: What works now
5. **CTA**: Follow the build, check the repo

## Image Style
- Terminal screenshots, code diffs, architecture sketches
- Dashboard showing real (not fake) progress
- Whiteboard or planning artifacts
